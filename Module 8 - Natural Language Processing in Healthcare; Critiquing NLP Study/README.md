## Critiquing NLP Study
Structured medical records are incomplete by design. Diagnostic codes capture what clinicians choose to document in a standardized format, but a significant portion of clinically relevant information lives in free-text notes, referrals, and patient histories that no code can fully represent. In kidney transplant patients, for example, graft loss has an ICD code, but its entry is not mandatory in most medical record systems, making accurate case identification unreliable through structured data alone. This module examined how natural language processing addresses that gap, how it actually works at a technical level, and what its limitations reveal about responsible implementation in clinical environments.

The NLP pipeline moves through deliberate stages before any model is trained. Raw clinical text is first cleaned, with HTML tags removed, spelling corrected, and abbreviations expanded. The cleaned text is then segmented and searched for clinically relevant terms. In identifying negative symptoms among schizophrenia patients, researchers built keyword lists across five symptom domains and applied Levenshtein distance algorithms to catch misspellings and synonyms that a direct keyword search would miss. Machine learning models are then trained on the extracted features and evaluated using metrics like F1 scores and AUC, which measure how well a model identifies true positives without overproducing false ones. Each stage matters because an error introduced early in the pipeline compounds through every step that follows.
The privacy dimension of NLP in healthcare is where responsible data science becomes non-negotiable. Training NLP models requires large datasets of sensitive patient information, and traditional systems were not built to handle that scale securely. Blockchain technology has emerged as one structural response to this problem. Its decentralized architecture eliminates single points of failure and its ledger ensures that every access or modification to patient data is recorded and traceable. When paired with homomorphic encryption, which allows machine learning operations to be performed directly on encrypted data without decryption, organizations can extract analytical value from sensitive datasets without exposing the underlying patient information. Smart contracts add another layer by defining exactly who can access what data and under what conditions.

The critical question this module raised was whether NLP tools should become a mandatory standard in hospital record systems or remain optional. That question does not have a clean answer. The clinical value is real. NLP surfaces information that documentation standards consistently fail to capture, and the patients most affected by those documentation gaps are often the ones with the most complex conditions. The barrier is not technical capability. It is the infrastructure, training, and governance required to deploy these tools responsibly at scale, and those requirements are not uniform across health systems. Mandating implementation without addressing those gaps would produce the same outcome as importing an unvalidated prognostic model into a setting it was never built for.
